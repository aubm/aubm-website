Git est un outil gestion de sources très sophistiqué que vous pouvez placer au centre de votre processus de build/ship.
Par l'utilisation de fonctionnalités comme les branches, les merges ou encore les hooks, Git vous laisse la main pour
contruire votre propre workflow en fournissant un maximum de flexibilité.

De mon point de vue, **il est tout aussi important de savoir utiliser Git de façon au moins aussi efficace que le(s) language(s)
que vous utilisez pour produire du code**.
Cet article va apporter les clés nécessaires pour bien utiliser Git, ainsi que quelques astuces pour aller un cran plus loin.
